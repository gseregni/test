Introduction

Pingendo is a Bootstrap builder available in three versions :  
Pingendo Playground. It's a free Bootstrap playground for the community, 
Pingendo Desktop.  It works locally  on filesystem  and offline,
Pingendo for Github. It works online integrated with Github 


Pingendo Playground

Pingendo Playground is an online Bootstrap HTML and SASS playground, available here  playground.pingendo.com
You can create, save and share your creations with others for free.
HTML and SASS will be saved as Gists on Github.

Pingendo Desktop

Pingendo Desktop it's an app for Windows (9+), OSX (10.9+) and Linux.  
You can get a copy here www.pingendo.com/desktop.html  

Once installed and launched you can create a page starting from a template. 
The result is saved as plain HTML, Saas and CSS files that you can modify also with external editors. No need of source export.
To learn more about what you can do in Pingendo jump to the Editor section. 

Pingendo for Github

Pingendo for Github it's a web app that works in Chrome or Safari. Support for other browsers will be added.
Integrates with Github API to load and save HTML pages and theirs assets stored in repositories.
Launching Pingendo for Guthub, it prompts you the Github login.  This is required in order to let Pingendo read and save files by the Github API.
We use standard OAuth mechanism provided by Github and we do not store in any way your password.
For additional info check out the Github website https://help.github.com/articles/connecting-with-third-party-applications/

Builder

The builder interface it's the same for all three versions and consists of 5 main areas 

1 Toolbar 

2 Sections and components panel. It permits you to browse and drag pre-made sections and Bootstrap 4 components

3 Canvas.
Here you can drop elements 

4 Property panel
Here you set  properties for the selection and for the page

5 HTML and Saas editor. Modifications are rendered live.


HTML

Pingendo let you build good-looking and perfectly coded HTML pages, deadly fast. 

To start mocking up a new page,  drag in the canvas sections and components.
Pre-made sections are the ones commonly used in site development.  Components are all those featured by Bootstrap 4.                              

Click on canvas to select the element and play with its properties using the selection tab of the property panel.  
Properties are the ones featured by Bootstrap. 

When an element is selected you can select its parent using the breadcrumb.
This is particularly useful when you can't pick an element directly from canvas.
You can also use the ESC key to select the parent.

If you are working on something more than a quick mockup, you can get your hand dirty using the HTML editor in the bottom HTML tab.

The editor is live, so the canvas will update almost realtime to reflect changes and vice versa. 
Since what you are editing is a live HTML DOM, you should  use Pingendo only on proper and valid HTML  documents. 
Preprocessors tags (like PHP) or templates engines are not supported and can't be saved correctly.


Sass

Sass is the CSS preprocessor used by Bootstrap 4.  
Differently by most Boostrap-based editors Pingendo  customize directly the Sass source and not simply the CSS in order to generate clean sources easily readable by developers .

You can play with most common Sass variables,  such as color scheme and fonts, from the  the page inspector.  If you need to customize more  you can edit the Sass source from the Sass tab bottom.

When you create a new page the  default Boostrap 4 theme will be attached by default.
As soon as you modify something you will be prompted to  give it a name and save it.

To check which is the Sass theme attached to a page go to the "PAGE" tab and look at the "FILE" property.

To attach an existing theme to a page click on the "FOLDER" icon of the file row and select the desired theme. 

Github integration

This part applies only to Pingendo for Github users. Pingendo Desktop users should  use instead Github Desktop client,

Pingendo for Github features Github integration to let you work online on pages stored in Github repositories.
Scope of the integration is to make simple for teams of developers and non developers to efficently collaborate building static web sites.
To do this Pingendo for Github offer an alternative and simplified version of Github web interface.

Pingendo for Gihthub use the same url scheme of Github. So if you need to access for example the repository testof exampleuser  you can point your browser to github.pingendo.com/exampleuser/test and you'll get the same as github.com/exampleuser/test.

The interface for working with the files in a repo is divided in 3 main areas

1 Current repository browser permit to switch between master version of your project and proposed revisions (Github pull requests)                                     
2 Browser for files contained in the current version 

3 Current file preview 

Pull requests

Pull requests let you tell others about changes you want to push to the master version. 
Once a pull request is sent, interested parties can review the set of changes, discuss potential modifications, and even push follow-up commits if necessary.

To create a new pull request click the "New pull request" button and give it a name that well describe what your changes are about.
Once created you'll get a fresh copy of the master version to work with safely.

As soon as you modify or delete existing files or upload a new one, a status badge will appear beside the file name. This to keep track easily of the changes contained in a pull request.

If you want  just discard a pull request use the close command. 
Instead if a pull request is ok and you  are one of the collaborators of the 
project you can merge it to the master version.

Warning, due to the way of working of Github Pull Request may be not mergeable sometimes.
This happens when a pull request modify a part of a file that has been modified also in the master version.


 

Github Pages

Github Pages it's a free hosting service kindly offered by Github. 
Master version of every repository can be publish to Github pages and so available on the internet.

To publish the master version use the publish button. Once publish you can visit the web site pointing your browser to <username>.github.io/<project>.


Permissions

Repository permissions are configured on Github. 
So, say we are working on a project named test. if you add userA as collaborator to test, userA will be able to merge and close pull request as well as editing the master version.
If instead userA is not part of the collaborator of test he will still be able to submit new pull-requests but he won't be able to merge them or edit anything else on the repo.
